# Runbooks

Практические руководства «что делать, когда…». Не теория, а пошаговые рецепты.

## Список

| Runbook | Когда |
|---|---|
| [`local-development.md`](local-development.md) | первый запуск проекта локально |
| [`testing.md`](testing.md) | запуск тестов |
| [`debugging.md`](debugging.md) | странное поведение, нужно понять root cause |
| [`release.md`](release.md) | подготовка релиза / hotfix |
| [`troubleshooting.md`](troubleshooting.md) | пользователь жалуется — как помочь |
| [`har-collection.md`](har-collection.md) | сбор HAR-трафика приложения для reverse engineering |

## Принципы

- Runbook = пошаговая инструкция, исполняемая копированием команд.
- Не дублирует методологию — ссылается на неё.
- Обновляется, когда меняется реальный flow.
