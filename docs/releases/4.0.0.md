# v4.0.0 — Домофон действительно стал частью умного дома

**Это тот самый большой релиз.** Раньше интеграция показывала камеры и открывала
двери. Теперь Home Assistant получает полный сценарий домофона: заметить
входящий звонок, показать гостя, принять вызов, поговорить, открыть дверь и
сохранить событие в истории. А камеры впервые можно стабильно отдать во внешний
RTSP — для NVR, Frigate, VLC или другой системы в вашей сети.

| | Главное |
|---|---|
| 📞 **Живой домофон** | FCM-звонок без polling, готовый экран вызова, SIP-ответ и двусторонний звук прямо в Home Assistant |
| 🕘 **Дом всё помнит** | Единая история принятых и пропущенных вызовов по местам, устройствам и аккаунтам |
| 📡 **Камеры вне карточки HA** | Opt-in публикация включённых камер по стабильному RTSP с keep-warm и отдельной настройкой hidden cameras |

> ✅ **Без breaking changes.** Существующие config entries продолжат работать,
> повторная авторизация и перенастройка устройств не нужны. Внешний RTSP
> выключен по умолчанию и включается только вручную.

## Что теперь умеет интеграция

- **Звонить в Home Assistant без polling.** Входящий вызов приходит через FCM
  практически сразу и доступен интерфейсу, уведомлениям и автоматизациям.
- **Отвечать и разговаривать.** SIP-регистрация создаётся только на время
  звонка; можно принять или завершить вызов и говорить с гостем через микрофон
  браузера.
- **Показывать весь вызов одним экраном.** Карточка
  `custom:eg-intercom-call-card` объединяет видео, звук, ответ, микрофон и
  защищённое открытие двери. Она адаптируется от телефона до настенной панели.
- **Хранить историю событий.** `custom:eg-event-history-card` собирает
  принятые и пропущенные вызовы, фильтрует их по устройствам и объединяет
  несколько мест или аккаунтов в одну временную ленту.
- **Публиковать камеры для внешних клиентов.** Включённые камеры получают
  стабильные RTSP-адреса через go2rtc; отдельная опция управляет hidden cameras,
  а disabled entities никогда не публикуются.
- **Строить автоматизации вокруг звонка.** Новые event entities и
  `sensor.*_call_state` дают стадии `idle`, `ringing`, `connecting`, `active`,
  `ended`, `error`. Можно показать экран, отправить уведомление или открыть
  дверь по собственному сценарию.
- **Безопасно делить один вызов между клиентами.** Несколько открытых экранов
  переиспользуют один call stream и не пересобирают его конкурентно.

## Исправлено

- Карточка истории больше не выводит внутренние значения Lit-шаблона
  `Обновить Обновить false` вместо пустого состояния или списка событий.
- Durable-history entities с событиями `call_accepted`/`call_missed` больше не
  объявляют несовместимый `device_class: doorbell`; realtime entity вызова
  сохраняет doorbell-класс и события `ring`/`ended`.
- History dispatcher изолирован по config entry, а baseline ведётся отдельно
  для каждого места и домофона. Добавление нового источника не воспроизводит
  его старые записи как новые события.
- Частичный сбой при обновлении общей карточки больше не удаляет уже
  загруженные строки временно недоступного аккаунта или места.
- Исправлена последовательность SIP pre-answer:
  `REGISTER → INVITE → 100 Trying`, а `200 OK` отправляется только после
  нажатия «Принять».
- Новый звонящий корректно заменяет предыдущий held-вызов; активный разговор
  новым звонком не прерывается.
- Устранены преждевременные пересборки call-video и EOF при нескольких
  одновременных зрителях.
- Запоздавший FCM `ended` от другого домофона больше не завершает текущий
  вызов и не оставляет карточку в неверном состоянии.
- FCM listener получил reconnect/watchdog и более надёжный cleanup.

## Клиентский профиль API

- Эмулируемый Android User-Agent обновлён до клиентского профиля `9.9.0`
  (`90900020`). Это снижает риск отказа API из-за устаревших метаданных клиента.
- Запрос live-видео и регистрация push-устройства синхронизированы с актуальным
  API-контрактом. Парольная и SMS-авторизация остаются совместимыми; повторная
  настройка существующих config entry не требуется.

## Внешний RTSP через go2rtc

- Новая опция **«Публиковать включённые камеры для внешнего RTSP»** создаёт
  стабильные go2rtc-потоки для enabled camera entities. По умолчанию она
  выключена и не меняет поведение существующих установок.
- Опция **«Также публиковать скрытые камеры»** отдельно добавляет hidden camera
  entities и доступна только вместе с основной публикацией. Disabled entities
  не публикуются.
- Каждый опубликованный поток сразу получает dedicated preload consumer,
  поэтому одноразовый operator source начинает использоваться до истечения.
  Источник обновляется каждые **28:30** PATCH-ом без замены активного consumer.
- Потерянные stream/preload/producer восстанавливаются reconcile-циклом после
  сбоя или перезапуска go2rtc. Изменение publication options применяется без
  полного reload интеграции и без обнуления уже работающих producers.
- Явное открытие enabled hidden camera в Home Assistant остаётся on-demand и
  работает без фонового preload, даже если hidden publication выключена.
- Диагностическая сущность **«Опубликованные RTSP-потоки»** показывает только
  фактически готовые адреса и состояние; credentials и operator source URL в
  атрибуты, diagnostics и логи не попадают. go2rtc Basic Auth поддерживается.

Интеграция публикует потоки только внутри настроенного go2rtc и не открывает
сетевые порты автоматически. Безопасный доступ к RTSP с других устройств или
сетей необходимо настроить самостоятельно через firewall/VPN и параметры
go2rtc.

## История событий

- Добавлена durable-история через API сервиса: отдельные event-сущности фиксируют
  новые принятые/пропущенные вызовы и motion для доступных intercom/public
  камер. Первый poll устанавливает тихий baseline, поэтому старые события не
  запускают автоматизации после обновления.
- Motion-history entities отключены по умолчанию. API polling камеры начинается
  только после включения соответствующей entity в Home Assistant.
- Дедупликация переживает перезапуск Home Assistant; сохраняются только
  ограниченные списки непрозрачных event ID. Текст сообщения оператора и
  другие неиспользуемые поля ответа в HA не переносятся.
- Старые вызовы можно просматривать через авторизованный entity-scoped
  WebSocket без воспроизведения их как новых событий или automation triggers.
  Для каждого места с домофоном создаётся отдельная history entity на
  существующем устройстве адреса; её default `entity_id` содержит `account_id`
  и `place_id`. Несколько мест и аккаунтов объединяются карточкой; прежние
  per-device history entity сохранены.

## Установка карточки

Интеграция уже содержит собранный JavaScript bundle. Добавьте Lovelace resource:

```text
URL: /elektronny_gorod_static/eg-intercom-call-card.js
Type: JavaScript Module
```

Пример конфигурации и требования к HTTPS/go2rtc:
[call-card-install.md](../features/intercom-two-way-audio/call-card-install.md).

Тот же resource содержит карточку истории:

```yaml
type: custom:eg-event-history-card
entity: event.account_123456_place_7890_event_history
```

Подробности: [history-card.md](../features/mobile-app-parity/history-card.md).

## Security

- Новых действий с токенами после обновления не требуется.
- Diagnostics продолжает маскировать access/refresh tokens, SIP/go2rtc
  credentials и PII.
- Микрофон доступен только авторизованным пользователям Home Assistant и
  только во время активного вызова.
- Backend `message` из истории отбрасывается на API boundary и не попадает в
  entity attributes, recorder, diagnostics, logs или persistent storage.
- Browse-команда проверяет `POLICY_READ` выбранной EventEntity и возвращает
  только opaque event ID, mapped type, timestamp и безопасную идентификацию
  источника (`place_id`, `source_id`, локальное имя из coordinator data).

## Breaking changes

Нет. `ConfigFlow.VERSION` остаётся `3`; миграция config entry не требуется.

## Известные ограничения

- Одновременно поддерживается один активный разговор на одну config entry.
- Для микрофона нужен HTTPS secure context.
- Браузер может потребовать один пользовательский жест, чтобы разрешить
  микрофон или запустить звук.
- Звук вызова и объединённый call stream требуют настроенного go2rtc.
- Внешняя RTSP-публикация требует доступных go2rtc HTTP API и RTSP-порта;
  функция выключена по умолчанию.
- Просмотр журнала принятых/пропущенных вызовов реализован; воспроизведение и
  скачивание связанных архивных clips через Media Source пока не входят в slice.

## Как обновиться

1. Установите 4.0.0 через HACS и перезапустите Home Assistant.
2. Повторно добавлять интеграцию или проходить авторизацию не нужно.
3. Для экрана вызова добавьте Lovelace resource из раздела выше и разместите
   `custom:eg-intercom-call-card` на dashboard.
4. Если нужен внешний RTSP, откройте настройки интеграции go2rtc и включите
   **«Публиковать включённые камеры для внешнего RTSP»**. Hidden cameras
   подключаются второй, отдельной опцией.

Если после обновления понадобится откат, выключите новые opt-in возможности и
вернитесь на предыдущий релиз через HACS. Миграции config entry и необратимых
изменений данных в 4.0.0 нет.
