# v4.0.0 — Звонки и двусторонняя связь с домофоном

Это крупнейшее обновление интеграции: «Электронный город» переходит от
камер, замков и периодического обновления к полноценному realtime-клиенту
домофона в Home Assistant.

## Что нового

- Входящие вызовы приходят в Home Assistant по FCM без polling.
- Добавлены event-сущность звонка и `sensor.*_call_state` со стадиями
  `idle`, `ringing`, `connecting`, `active`, `ended`, `error`.
- Реализован SIP-приём вызова: регистрация при звонке, ответ, отклонение,
  завершение и корректная обработка `CANCEL`/`BYE`.
- Видео и звук гостя объединены в `camera.intercom_call` через HA-native
  WebRTC/HLS; go2rtc остаётся внутри локальной сети.
- Добавлен двусторонний звук: микрофон браузера передаётся через
  авторизованный Home Assistant WebSocket в SIP/RTP домофона.
- Готовая Lovelace-карточка `custom:eg-intercom-call-card`: видео,
  принять/завершить вызов, открыть дверь, включить микрофон и звук.
- Добавлены blueprints и примеры dashboard для уведомления и экрана звонка.
- Один активный поток вызова безопасно переиспользуется несколькими клиентами;
  конкурентные открытия больше не пересобирают operator stream.

## Исправлено

- SIP pre-answer приведён к поведению штатного Android-приложения:
  `REGISTER → INVITE → 100 Trying`, а `200 OK` отправляется только после
  нажатия «Принять».
- Новый звонящий корректно заменяет предыдущий held-вызов; активный разговор
  новым звонком не прерывается.
- Устранены преждевременные пересборки call-video и EOF при нескольких
  одновременных зрителях.
- Запоздавший FCM `ended` от другого домофона больше не завершает текущий
  вызов и не оставляет карточку в неверном состоянии.
- FCM listener получил reconnect/watchdog и более надёжный cleanup.

## Совместимость с мобильными приложениями 9.9.0

- Интеграция теперь представляется актуальной версией клиента «Мой Дом» и
  «Умный Дом.ру» — `9.9.0` (`90900020`). Это снижает риск отказа API из-за
  устаревшей версии мобильного клиента.
- Запрос live-видео и регистрация push-устройства приведены к фактическому
  поведению «Мой Дом» 9.9.0. Парольная и SMS-авторизация остаются совместимыми;
  повторная настройка существующих config entry не требуется.

## Установка карточки

Интеграция уже содержит собранный JavaScript bundle. Добавьте Lovelace resource:

```text
URL: /elektronny_gorod_static/eg-intercom-call-card.js
Type: JavaScript Module
```

Пример конфигурации и требования к HTTPS/go2rtc:
[call-card-install.md](../features/intercom-two-way-audio/call-card-install.md).

## Security

- Новых действий с токенами после обновления не требуется.
- Diagnostics продолжает маскировать access/refresh tokens, SIP/go2rtc
  credentials и PII.
- Микрофон доступен только авторизованным пользователям Home Assistant и
  только во время активного вызова.

## Breaking changes

Нет. `ConfigFlow.VERSION` остаётся `3`; миграция config entry не требуется.

## Известные ограничения

- Одновременно поддерживается один активный разговор на одну config entry.
- Для микрофона нужен HTTPS secure context.
- Браузер может потребовать один пользовательский жест, чтобы разрешить
  микрофон или запустить звук.
- Звук вызова и объединённый call stream требуют настроенного go2rtc.

## Обновление

Обновитесь через HACS и перезапустите Home Assistant. Полный список изменений —
в секции `[Unreleased]` файла `CHANGELOG.md`; при публикации она будет отмечена
как `4.0.0`.
