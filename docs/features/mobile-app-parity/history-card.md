# Карточка истории вызовов

`custom:eg-event-history-card` показывает общую ленту предыдущих принятых и
пропущенных вызовов из журнала приложения 9.9.0. Это отдельный browse-интерфейс:
открытие карточки не изменяет EventEntity и не запускает автоматизации.

## Подключение

Добавьте один Lovelace resource типа **JavaScript Module**:

```text
/elektronny_gorod_static/eg-intercom-call-card.js
```

Тот же bundle содержит и экран активного вызова, и карточку истории. После
обновления ресурса может потребоваться hard refresh браузера.

Для каждого места с домофоном интеграция создаёт entity «История событий» и
привязывает её к существующему HA-устройству адреса. Default `entity_id`
содержит ID аккаунта и места, поэтому одинаковые адреса разных договоров не
смешиваются. Добавьте нужную entity в карточку:

```yaml
type: custom:eg-event-history-card
entity: event.account_123456_place_7890_event_history
```

Если у аккаунта несколько мест или в Home Assistant настроено несколько
аккаунтов/договоров, их place-history entities можно объединить в одну
хронологическую ленту. Каждая backend-страница загружается независимо, поэтому
более короткая история одного места не останавливает остальные:

```yaml
type: custom:eg-event-history-card
entities:
  - event.account_123456_place_7890_event_history
  - event.account_123456_place_7891_event_history
  - event.account_654321_place_9876_event_history
title: Все события
```

Для обратной совместимости можно передать прежнюю entity «История вызовов
домофона»: тогда карточка покажет только выбранный `access_control_id`. Нельзя
использовать realtime entity активного звонка или camera-motion history entity.

## Поведение

- первая страница загружается при открытии карточки;
- place entity запрашивает только свой `placeId`;
- события из нескольких мест и аккаунтов объединяются по времени без коллизий
  opaque event ID;
- фильтр «Устройства» использует составной ключ config entry + place + access
  control и не смешивает одноимённые домофоны разных аккаунтов;
- «Показать ещё» читает следующую backend-страницу и убирает дубли по opaque
  event ID;
- «Обновить» заново читает page 0;
- ошибка текущего backend-запроса показывается как временная недоступность;
  повтор выполняется по кнопке «Обновить», а poller попробует на следующем tick;
- даты и время форматируются в локали Home Assistant;
- backend-текст `message`, subscriber/account data и другие поля в браузер не
  передаются.

WebSocket-команда проверяет право текущего HA-пользователя на чтение каждой
указанной EventEntity и жёстко связывает запрос с её config entry. Place entity
разрешает только access controls своего места; per-device entity дополнительно
ограничивает ответ одним домофоном. Произвольный ID не даёт доступ к другому
месту или аккаунту.

## Ограничения

- отображаются только подтверждённые capture-ом типы: принятый и пропущенный
  вызов;
- старые строки доступны в карточке, но намеренно не replay-ятся в нативную
  Recorder History; там появляются только новые события после baseline;
- UI ограничивает запрос страниц диапазоном `0..100`;
- воспроизведение архивного видео по строке события пока не реализовано — это
  отдельный Media Source slice.
