# Визуальный аудит шаблонности

Проведён 2026-07-19 в реальном браузере (Playwright, сборка `vite preview`):
desktop 1440×900, mobile 390×844, светлая и тёмная темы, консоль без ошибок.

## Вопросы контроля

**Можно ли заменить название проекта на любой SaaS и ничего не изменится?**
Нет. Главный объект каждого ключевого экрана — работающая карточка вызова
домофона с видео гостя, слайдером «Открыть» и SIP-фазами. Hero, сценарий и
playground не имеют смысла вне этого продукта.

**Есть ли секция, которую невозможно спутать с шаблоном?**
Три: (1) hero с автосценарием «путь одного звонка» — реальный интерфейс
проживает звонок по циклу с моноширинным статус-логом и картой сигнальных
узлов; (2) скролл-повествование, где липкая сцена переключает настоящие фазы
карточки; (3) playground, в котором «Принять» действительно запускает
connecting→active через перехваченный сервис-вызов.

**Читается ли идея домофона и умного дома без текста?**
Да: видео гостя у панели домофона, кнопка «Открыть» с замком, карта
«подъезд → push → ha → устройства → замок → история».

**Есть ли у сайта собственный характер?**
Сигнальная нить вдоль страницы, Unbounded в заголовках, статус-строки
`lock.unlock → дверь открыта` как фирменный приём, глубокая тёмная сцена для
видео-контекста при светлой странице.

**Есть ли повторяющиеся карточки, которые стоило заменить?**
Секция возможностей сделана редакционным списком с линейками и индексами
(не сеткой одинаковых карточек с иконками). Карточки остались только там, где
это содержательно оправдано: рецепты автоматизаций (у каждого — сложность,
сущности, предупреждения, YAML) и вердикты совместимости.

**Не выглядит ли сайт как один промпт в генераторе лендингов?**
Анти-паттерны из брифа отсутствуют: нет фиолетово-синих градиентов,
glassmorphism, светящихся шаров, иконок в цветных кружках, трёх карточек
«быстро/удобно/безопасно», bento-сеток. Hero асимметричный, живой интерфейс
вместо мокапа.

**Достаточно ли проработаны детали?**
Hover/focus-состояния на всех интерактивах, видимый `:focus-visible`,
`prefers-reduced-motion` (полное отключение анимаций, статичный кадр hero),
фоллбек скриншотом при сбое загрузки демо-бандла, пустые/ошибочные состояния
в playground, лог с автопрокруткой и лимитом строк.

**Создаёт ли первый экран желание взаимодействовать?**
Первое, что видно, — гость звонит в дверь и интерфейс отвечает; CTA
«Посмотреть, как работает» ведёт в повествование, «Установить через HACS» —
прямой путь.

## Найдено и исправлено в ходе аудита

- Подписи узлов сигнальной карты пересекались на desktop → короткая метка `ha`.
- Навигация шапки схлопывалась на 390px → перенос в отдельную строку с
  горизонтальной прокруткой.
- В headless-браузере активная фаза показывала permission-баннер микрофона →
  в симуляции доступ помечается granted (микрофон в демо не используется).
